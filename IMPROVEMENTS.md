# One UI — улучшения, оптимизации и потенциальные баги

Разбор по состоянию репозитория на июнь 2026. Всё ниже основано на реальном коде
и на замерах бандла, выполненных одинаковой методикой (esbuild, min + gzip, React
вынесен во external). Пункты отсортированы примерно по соотношению «эффект / усилие».

> **Уже применено в этом проходе** (без коммита): `sideEffects: ["**/*.css"]` в
> `package.json`; исправлен дубликат `jelly-odd` (теперь зеркальная фаза — соседние
> тосты колышутся по-разному); no-JS fallback для `.reveal` в `layout.tsx`; SEO —
> `robots.ts`, `sitemap.ts`, OpenGraph/Twitter/`metadataBase`; `@storybook/addon-a11y`;
> CI-workflow (`tsc` + `lint` + `build` + `build-storybook`).
>
> **Осознанно отложено:** массовый перевод обёрток на `forwardRef` (на React 19, который
> требует пакет, паттерн `render` и так работает — не оправдывает правку 51 файла);
> переход на локальный шрифт (нужны бинарники woff2; на машине с интернетом текущий
> вариант рабочий); полный `exports`-map + сборка пакета (этап публикации); финальное имя
> пакета (решение за тобой).

---

## TL;DR — что сделать в первую очередь

1. Подготовить пакет к публикации: `sideEffects`, `exports`-map, `peerDependencies`,
   ESM-сборка с типами. Это одновременно и условие публикации, и крупнейший рычаг
   уменьшения бандла у потребителя.
2. Сделать обёртки (`Button`, `Input`, `Chip`, `Card`, …) через `forwardRef` — сейчас
   ни одна из них не пробрасывает ref, а паттерн `render={<Button/>}` на это полагается.
3. Убрать дубликат `jelly` / `jelly-odd` и самостоятельно хостить шрифт вместо
   `next/font/google` (сейчас сборка требует доступ к сети).

## 1. Упаковка для npm (главный рычаг)

Сейчас это `private: true` Next-приложение, а не публикуемый пакет: нет `exports`,
нет сборки библиотеки, нет генерации `.d.ts`. Пока это не починено, цифры «вес одного
компонента» достижимы только при идеальном tree-shaking у потребителя — а его легко
сломать.

**Что добавить в `package.json`:**

```jsonc
{
  "name": "one-ui",              // см. ниже про имя
  "type": "module",
  "sideEffects": false,          // КЛЮЧЕВОЕ: компоненты не импортируют css —
                                 // можно безопасно отбрасывать неиспользуемое
  "exports": {
    ".": { "types": "./dist/index.d.ts", "import": "./dist/index.js" },
    "./button": { "types": "./dist/button.d.ts", "import": "./dist/button.js" },
    "./styles.css": "./dist/styles.css"
    // …по точке входа на компонент
  },
  "files": ["dist"],
  "peerDependencies": {
    "react": ">=19",
    "react-dom": ">=19",
    "@base-ui/react": "^1.6.0"
  }
}
```

- **`"sideEffects": false`** — проверено: ни один `*.tsx` в `src/ui/components` не
  импортирует css (стили собраны централизованно в `styles.css`). Значит флаг безопасен
  и позволяет бандлерам выкидывать неиспользуемые ветки барреля `components/index.ts`.
- **`exports` с точками входа на компонент** — чтобы `import { Button } from "one-ui"`
  не рисковал затащить Base UI-зависимости соседних компонентов через барель. Идеал —
  `import { Button } from "one-ui/button"`.
- **`peerDependencies`** для `react`, `react-dom`, `@base-ui/react`, `lucide-react` —
  чтобы они не дублировались в бандле потребителя.
- **Сборка**: `tsup` или Rollup, ESM-only, с `dts`. Отдельной командой собирать CSS
  (Tailwind v4) в `dist/styles.css`.
- **Имя**: в `package.json` стоит `one-ui.v2`, а команда установки на сайте — `one-ui`.
  Нужно выбрать финальное имя и проверить занятость на npm (учитывая конфликт с Samsung
  One UI — возможно, нужен scope, напр. `@yourorg/one-ui`).

## 2. Дальнейшее уменьшение бандла

Замеренные размеры (кБ, gzip, React внешний):

| Компонент | One UI | Base UI | Комментарий |
|---|---|---|---|
| Button | 3.5 | 3.0 | обёртка ≈ +0.5 кБ |
| Switch | 6.3 | 6.0 | +0.3 кБ |
| Dialog | 24.0 | 22.4 | вес в примитиве Base UI |
| Select | 45.2 | 43.8 | самый тяжёлый |
| Combobox | 50.9 | — | + фильтрация/чипы |
| Вся библиотека (без tree-shaking) | 171.5 | | барель целиком |

Вывод: **почти весь вес — это примитивы Base UI, а не обёртки One UI.** Поэтому
основные рычаги — не «писать меньше кода в обёртках», а не давать затащить лишнее:

1. **Tree-shaking барреля** (`sideEffects: false` + per-component `exports`) — самый
   большой эффект. Без него `import { Button }` может утянуть граф зависимостей всех 51
   компонента (до ~171 кБ) вместо 3.5 кБ.
2. **Иконки**: компоненты импортируют конкретные иконки (`import { Check } from
   "lucide-react"`) — это уже tree-shakeable по иконке. Но стоит зафиксировать lucide во
   `peerDependencies` и проверить версию (см. п.3 — `^1.21.0` подозрительно высокая).
3. **Тяжёлые оверлеи** (Select 45, Combobox 51, Tooltip 34, Dialog 24) тянут движок
   позиционирования (Floating UI) из Base UI. В приложении их стоит грузить лениво
   (`next/dynamic` / `React.lazy`) — они почти всегда не нужны на первом экране.
4. **ESM-only, без CJS** — двойная сборка CJS/ESM мешает агрессивному tree-shaking.
5. **CSS**: Tailwind v4 сам отбрасывает неиспользованные классы при сканировании кода
   потребителя. Убедиться, что в гайде по установке указано подключать `styles.css` один
   раз и что content-сканирование настроено — иначе попадёт весь слой утилит.
6. **Опциональный «lite»-Tooltip** без `Portal`/`Arrow` для простых подсказок — экономит
   заметную долю от 34 кБ там, где не нужно портирование и стрелка.

> Перепроверять в любой момент: `npm run measure` (скрипт `scripts/measure-bundle.mjs`).

## 3. Потенциальные баги и риски корректности

### 3.1. Обёртки не пробрасывают `ref` (высокий приоритет)
Проверено: ни один компонент в `src/ui/components` не использует `forwardRef`. Обёртки
вида `Button = (props) => <BaseButton {...props}/>` работают при прямом использовании, но
паттерн `render={<Button/>}` (я применяю его в `Tooltip.Trigger`, `Field.Control`) опирается
на проброс ref в триггер. Сейчас это «вывозит» только за счёт поведения React 19
(ref-as-prop). На React 18 или при строгой проверке это даст предупреждение и поломанное
позиционирование. **Фикс:** обернуть публичные обёртки в `forwardRef` и прокидывать `ref`
в Base UI-примитив.

### 3.2. `next/font/google` требует сеть при сборке
`layout.tsx` грузит Geist через `next/font/google`. Сборка в окружении без доступа к
`fonts.googleapis.com` падает (наблюдалось в песочнице: «Error while requesting resource»).
**Фикс:** самостоятельный хостинг шрифта (`next/font/local`) или WOFF2 в `public/`.

### 3.3. Дубликат keyframes `jelly` / `jelly-odd`
Проверено: в `src/ui/styles/animations.css` блоки `@keyframes jelly` и `@keyframes
jelly-odd` идентичны байт-в-байт. Либо `jelly-odd` задумывался зеркальным (тогда это баг
— анимация не отличается), либо это мёртвый код. **Фикс:** сделать `jelly-odd` реально
другим (например, инверсная фаза сжатия) или удалить.

### 3.4. Контент скрыт до выполнения JS (scroll-reveal)
`Reveal` стартует с `opacity: 0` и показывает контент по `IntersectionObserver`. При
отключённом JS или ошибке гидрации часть страницы останется невидимой; это также минус
для SEO-краулеров без JS и для доступности. **Фикс:** дефолтить видимость и только
*прятать* при наличии JS (`<noscript>`-стиль или класс на `<html>`), либо помечать
`reveal` только после маунта.

### 3.5. Версия `lucide-react ^1.21.0`
Официальный `lucide-react` сейчас в линейке `0.x`; мажор `1.21.0` — это либо форк, либо
зеркало. На этом и «погорела» иконка `Github` (удалена из набора — я заменил на `Star`).
**Фикс:** зафиксировать, что это за пакет, и проверить наличие нужных иконок при апгрейдах.

### 3.6. Рассинхрон имени пакета
`one-ui.v2` в `package.json` против `one-ui` в команде установки на сайте и в доках.
Выбрать одно и прогнать заменой по `registry.tsx`, `page.tsx`, гайдам.

### 3.7. Мелочи
- Барель `components/index.ts` реэкспортит всё — без `sideEffects: false` это анти-паттерн
  для tree-shaking (см. п.1–2).
- В доках (`/docs`) сложные оверлеи показаны как «API-карточка», а не живьём — не баг, но
  стоит добавить интерактивные примеры (см. п.5).

## 4. Производительность и DX

- **Ленивая загрузка оверлеев** в приложениях (см. 2.3) — меньше JS на первом экране.
- **`prefers-reduced-motion`**: в `landing.css` учтён; проверить, что и библиотечные
  анимации (jelly на кнопках) уважают `motion-reduce` (в `button.styles.css` есть
  `motion-reduce:transition-none` — хорошо, держать инвариант для новых компонентов).
- **OG-картинка, `sitemap`, `robots`** для лендинга и `/docs` — для шеринга и индексации.
- **Локализация**: тексты сайта на английском; если целевая аудитория и Habr — RU,
  предусмотреть i18n или вторую языковую версию.

## 5. Storybook и документация

- Сейчас 51 история генерируется из общего реестра; оверлеи отдают API-заметку, а не
  живой пример. Добавить полноценные интерактивные истории для `Dialog`, `Drawer`,
  `Select`, `Combobox`, `Menu` с `play`-функциями (`@storybook/test`).
- Подключить **`@storybook/addon-a11y`** — автопроверка контраста/ARIA по каждой истории.
- **Визуальная регрессия** (Chromatic) — особенно ценно для UI-кита: ловит «поехавшие»
  стили при апдейте Base UI/Tailwind.
- Страница `Bundle size` в Storybook использует зашитые данные из `bundle-data.ts`.
  Можно завести шаг CI, который гоняет `npm run measure` и обновляет JSON, чтобы цифры
  не устаревали.
